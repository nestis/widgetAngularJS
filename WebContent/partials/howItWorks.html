Widget AngularJS

How it works

First, a directive

The file WidgetDirective.js defines a directive (widget) to be used as an attribute. That means that when the DOM compilation is happening, each HTML element that has this attribute will be processed as a widget.
This process modifies the DOM to add a loading and error DIVs, initializes common variables for all the widgets, and in case the widget has specified it, it loads JavaScripts libraries.

Then, the controller

Once the directive has finished, it makes a call to WidgetController. At this stage, this controller only subscribes the $scope to a UI-tabs event. Each type of widget must prototype-inherit from WidgetController and define a function called processWidget. This function is where the logic to create the widget must be.

Yeah, the service

The DataService is invoked by the controller when it needs to make a request to the REST API. This Service makes the request and returns the promise object to the controller.

The final step!

The controller receives the promise object and defines the successCallback and errorCallback for it. If the request has been successful, the controller will proceed to render the widget with the data obtained.
