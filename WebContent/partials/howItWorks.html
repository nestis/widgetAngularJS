Widget AngularJS

How it works

First, a directive

The file directives.js defines a directive (widgetDefinition) placed in an attribute. That means that when the DOM compilation is happening, each HTML element that has that attribute will be processed as a widget.
This process modifies the DOM to add a loading and error DIVs, inits a commons variables for all the widgets and in case the widget needs it, it loads JavaScripts libraries.

Then, the controller

When the directive has finished, it made a call to the WidgetController. At this stage, the controller only evaluates the type of the widget and invokes the proper logic for it.

Yeah, the service

The DataService is invoked by the controller when it needs to make a request to the REST API. This Service makes the request and returns the promise object to the controller.

The final step!

The controller receives the promise object and defines the successCallback and errorCallback for it. If the request has been successful, the controller will proceed to create the widget with the data obtained.
